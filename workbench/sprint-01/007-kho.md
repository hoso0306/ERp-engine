# Task 05 — Stock Consistency Rule

## Mục tiêu

**Đây là task Review, không phải Implementation.** Việc cập nhật `Material.currentStock` đã được code trong Task 02 (Material Receipt) và Task 03 (Material Issue) — task này chỉ verify lại tính nhất quán, không viết thêm code mới.

Nguồn dữ liệu:

```text
WarehouseTransaction

↓

Material.currentStock
```

**Business Rule (Data Integrity):** `Material.currentStock` chỉ được cập nhật theo **Delta** của từng `WarehouseTransaction` (`IN` → `+= quantity`, `OUT` → `-= quantity`). **Không được tính lại toàn bộ lịch sử** (`SUM(WarehouseTransaction)`) trong bất kỳ luồng nào — kể cả khi sau này có thêm `Inventory Adjustment`. Lý do: giữ hiệu năng tốt khi dữ liệu tăng lên, và nhất quán với quyết định không lưu `runningBalance` (xem `warehouse.md`).

---

## Definition of Done

- [x] Verify: `currentStock` được cập nhật bằng Delta (`+=`/`-=`), không có chỗ nào tính lại bằng `SUM(WarehouseTransaction)`.
- [x] Verify: IN cộng, OUT trừ đúng.
- [x] Verify: Không âm kho.
- [x] Verify: Idempotent (xem Task 03 DoD).
